data-attr
Sets the value of any HTML attribute to an expression, and keeps it in sync. 
<div data-attr:title="$foo"></div>
Can also set multiple attributes using key-value pairs: 
<div data-attr="{title: $foo, disabled: $bar}"></div>
data-bind
Creates a signal and sets up two-way data binding between it and an element's value. The value of the element is updated when the signal changes, and the signal value is updated when the value of the element changes. 
<input data-bind:foo />
Works with input, select, textarea elements, and web components. Event listeners are added for change and input events.
data-class
Adds or removes a class to or from an element based on an expression. 
<div data-class:hidden="$foo"></div>
Can also manage multiple classes: 
<div data-class="{hidden: $foo, 'font-bold': $bar}"></div>
data-computed
Creates a signal that is computed based on an expression. The computed signal is read-only, and its value is automatically updated when any signals in the expression are updated. 
<div data-computed:foo="$bar + $baz"></div>
Useful for memoizing expressions containing other signals.
data-effect
Executes an expression on page load and whenever any signals in the expression change. Useful for performing side effects, such as updating other signals, making requests to the backend, or manipulating the DOM. 
<div data-effect="$foo = $bar + $baz"></div>
data-show
Shows or hides an element based on whether an expression evaluates to true or false. 
<div data-show="$foo"></div>
To prevent flickering before Datastar processes the DOM, add a display: none style initially.
data-signals
Patches (adds, updates or removes) one or more signals into the existing signals. Values defined later in the DOM tree override those defined earlier. 
<div data-signals:foo="1"></div>
Can also use key-value pairs: 
<div data-signals="{foo: {bar: 1, baz: 2}}"></div>
data-on
Attaches an event listener to an element, executing an expression whenever the event is triggered. 
<button data-on:click="$foo = ''">Reset</button>
An evt variable representing the event object is available in the expression. Supports modifiers like __debounce, __throttle, __prevent, __stop, and more.
data-text
Binds the text content of an element to an expression. 
<div data-text="$foo"></div>
data-style
Sets the value of inline CSS styles on an element based on an expression, and keeps them in sync. 
<div data-style:background-color="$usingRed ? 'red' : 'blue'"></div>
Can set multiple style properties: 
<div data-style="{display: 'flex', flexDirection: 'column'}"></div>
data-init
Runs an expression when the attribute is initialized. This can happen on page load, when an element is patched into the DOM, and any time the attribute is modified. 
<div data-init="$count = 1"></div>
Supports modifiers like __delay and __viewtransition.
data-ref
Creates a new signal that is a reference to the element on which the data attribute is placed. 
<div data-ref:foo></div>
The signal value can then be used to reference the element and access its properties.
data-indicator
Creates a signal and sets its value to true while a fetch request is in flight, otherwise false. Useful for showing loading indicators. 
<button data-on:click="@get('/endpoint')" data-indicator:fetching></button>
<div data-show="$fetching">Loading...</div>
data-on-intersect
Runs an expression when the element intersects with the viewport. 
<div data-on-intersect="$intersected = true"></div>
Supports modifiers like __once, __half, __full, __debounce, and __throttle.
